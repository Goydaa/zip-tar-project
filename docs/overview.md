# ZIPFILE и TARFILE - Полный обзор 
 
## Модуль 1. Контекст и предпосылки 
 
### 1. Ограничения стандартных средств 
Ручная работа с архивами через проводник или команды tar имеет ограничения: 
- Отсутствие автоматизации 
- Проблемы с кроссплатформенностью 
- Нет программного доступа к содержимому без распаковки 
 
### 2. Проблема, которую решает инструмент 
Модули zipfile и tarfile позволяют: 
- Создавать и читать архивы из Python-кода 
- Работать одинаково на Windows, Linux и macOS 
- Читать файлы из архива без распаковки на диск 
 
### 3. Инженерная грамотность 
Использование этих модулей формирует понимание: 
- Архивы как структурированные контейнеры 
- Важность выбора формата (ZIP vs TAR.GZ) 
- Безопасность при распаковке (path traversal) 
 
### 4. Применение в реальных проектах 
- Резервное копирование данных 
- Экспорт файлов для скачивания 
- Распаковка дистрибутивов ПО 
- Обработка сжатых логов 
 
## Модуль 2. Основные идеи и механизмы 
 
### 5. Центральные объекты 
- ZipFile - основной класс для ZIP-архивов 
- TarFile - основной класс для TAR-архивов 
- TarInfo - метаданные о файле в TAR-архиве 
 
### 6. Ключевые механизмы 
- Контекстный менеджер with для автоматического закрытия 
- Режимы: r (чтение), w (запись), a (добавление) 
- write() - добавление файла 
- extractall() - распаковка 
 
### 7. Работа с данными 
- namelist() - список файлов в архиве 
- read() - чтение файла без распаковки 
- getinfo() - метаданные файла 
 
### 8. Обработка ошибок 
- BadZipFile - поврежденный ZIP 
- LargeZipFile - архив больше 4ГБ 
- ReadError - ошибка чтения TAR 
- ExtractError - ошибка распаковки
