## МОДУЛЬ 1. КОНТЕКСТ И ПРЕДПОСЫЛКИ 
 
### 1. ОГРАНИЧЕНИЯ СТАНДАРТНЫХ СРЕДСТВ 
 
**Что обычно используют для работы с архивами:** 
 
- **Проводник Windows** - нужно вручную выделять файлы, нажимать "Сжать" 
- **Команда tar в Linux** - требует знания синтаксиса и не работает в Windows 
- **WinRAR/7-Zip** - хорошие программы, но ими нельзя управлять из кода 
 
**В чем их проблемы:** 
 
🔹 **Нет автоматизации** - нельзя встроить в скрипт, чтобы каждый день создавался бэкап 
🔹 **Не кроссплатформенные** - tar есть в Linux, но нет в Windows по умолчанию 
🔹 **Нельзя прочитать файл из архива без распаковки** - приходится сначала распаковывать всё на диск 
🔹 **Сложно работать с большим количеством архивов** - каждый архив нужно открывать вручную 
 
--- 
 
### 2. ПРОБЛЕМА, КОТОРУЮ РЕШАЮТ ZIPFILE И TARFILE 
 
Эти модули решают главную проблему: **автоматизация работы с архивами из Python-кода**. 
 
**Что теперь можно делать:** 
 
✅ **Создавать архивы программно** - написал скрипт, и он каждый день создаёт бэкап 
✅ **Читать содержимое без распаковки** - можно посмотреть список файлов и даже прочитать файл из архива 
✅ **Работать одинаково на всех ОС** - код на Python работает везде 
✅ **Добавлять файлы в существующий архив** - без перепаковки всего архива 
✅ **Сохранять метаданные** - даты создания, права доступа 
 
--- 
 
### 3. ИНЖЕНЕРНАЯ ГРАМОТНОСТЬ 
 
Использование этих модулей формирует важные инженерные навыки: 
 
**Понимание форматов архивов** 
- ZIP - лучше для максимальной совместимости 
- TAR.GZ - лучше для Unix-систем и лучшего сжатия 
- Нужно выбирать формат в зависимости от задачи 
 
**Безопасность при распаковке** 
- Нужно проверять пути в архиве, чтобы не было path traversal атаки 
- Например, архив может содержать файл `../../config/passwords.txt` 
 
**Компромиссы между скоростью и размером** 
- `ZIP_STORED` - без сжатия, быстро 
- `ZIP_DEFLATED` - со сжатием, медленнее, но архив меньше

### 4. ПРИМЕНЕНИЕ В РЕАЛЬНЫХ ПРОЕКТАХ 
 
🔥 **Системы резервного копирования** 
  - Ежедневно создают ZIP-архивы важных папок 
  - Хранят архивы за последние 30 дней 
 
🔥 **Экспорт данных из веб-приложений** 
  - Пользователь выбирает несколько файлов → сервер упаковывает их в ZIP → пользователь скачивает 
  - Например, экспорт всех фото из альбома 
 
🔥 **Распаковка дистрибутивов ПО** 
  - pip скачивает библиотеки в ZIP/TAR.GZ и распаковывает их 
  - Docker-образы тоже архивы 
 
🔥 **Обработка логов** 
  - Логи сжимаются в TAR.GZ для экономии места 
  - Скрипты читают логи прямо из архива, не распаковывая 
 
🔥 **CI/CD пайплайны** 
  - Артефакты сборки упаковываются в архивы 
  - Передаются между этапами сборки

## МОДУЛЬ 2. ОСНОВНЫЕ ИДЕИ И МЕХАНИЗМЫ 
 
### 5. ЦЕНТРАЛЬНЫЕ ОБЪЕКТЫ 
 
**ZipFile** - основной класс для работы с ZIP-архивами 
```python 
from zipfile import ZipFile 
with ZipFile('archive.zip', 'r') as zipf: 
    zipf.extractall('extracted/') 
``` 
 
**TarFile** - основной класс для работы с TAR-архивами 
```python 
import tarfile 
with tarfile.open('archive.tar.gz', 'r:gz') as tarf: 
    tarf.extractall('extracted/') 
``` 
 
**TarInfo** - метаданные о файле в TAR-архиве 
```python 
for member in tarf.getmembers(): 
    print(member.name, member.size, member.mtime) 
``` 
 
--- 
 
### 6. КЛЮЧЕВЫЕ МЕХАНИЗМЫ 
 
**Контекстный менеджер with** - автоматически закрывает архив 
```python 
with zipfile.ZipFile('archive.zip', 'w') as zipf: 
    zipf.write('file.txt')  # архив закроется сам после выхода из блока 
``` 
 
**Режимы работы:** 
- `'r'` - чтение 
- `'w'` - запись (создание нового) 
- `'a'` - добавление (только для ZIP) 
- `'x'` - эксклюзивное создание 
 
**Методы ZipFile:** 
- `write(filename, arcname)` - добавить файл 
- `extractall(path)` - распаковать всё 
- `extract(member, path)` - распаковать один файл 
- `read(name)` - прочитать файл без распаковки 
- `namelist()` - получить список всех файлов 
- `infolist()` - получить список с метаданными

### 7. РАБОТА С ДАННЫМИ 
 
**Чтение файла из архива без распаковки:** 
```python 
with zipfile.ZipFile('archive.zip', 'r') as zipf: 
    content = zipf.read('config.json')  # читаем в память 
    data = json.loads(content)  # парсим JSON 
``` 
 
**Добавление данных из памяти (без создания временного файла):** 
```python 
import io 
with zipfile.ZipFile('archive.zip', 'w') as zipf: 
    zipf.writestr('hello.txt', 'Hello, World!') 
``` 
 
**Сохранение метаданных:** 
```python 
import time 
with zipfile.ZipFile('archive.zip', 'w') as zipf: 
    info = zipfile.ZipInfo('file.txt') 
    info.date_time = time.localtime() 
    zipf.writestr(info, 'content') 
```

### 8. ОБРАБОТКА ОШИБОК 
 
**zipfile исключения:** 
- `BadZipFile` - файл не ZIP или повреждён 
- `LargeZipFile` - архив больше 4ГБ без allowZip64 
 
**tarfile исключения:** 
- `ReadError` - архив не может быть прочитан 
- `CompressionError` - ошибка распаковки 
- `ExtractError` - ошибка извлечения 
 
**Пример обработки:** 
```python 
try: 
    with zipfile.ZipFile('bad.zip', 'r') as zipf: 
        zipf.extractall('out/') 
except zipfile.BadZipFile: 
    print("Файл повреждён или не является ZIP") 
```
